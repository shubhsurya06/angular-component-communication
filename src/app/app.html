Parent Child Communication - TWEEKS when OBJECT value changes - with ngOnChanges and @Input()

ngOnChanges will detect changes in primitive data types like string, number, boolean. But for complex data types like object and array, it will not detect the changes unless the reference of the object/array is changed.

In this example, we are chaning original parent object, so its referece gets changed and ngOnChanges in child component is triggered.

If we were to change a property of the object without changing the reference, ngOnChanges in child component would NOT be triggered. Go to app.ts file and uncomment line no 34 and comment rest of the lines to see the difference.
